## Purpose

This document captures reusable engineering practices across projects and serves as the default standard as well as the source for shareable standard packages.

## Scope

- Applies to every project that installs or adopts the shared development standard.
- If a project does not override a case explicitly, this document is the default.
- If a project has a valid project-specific override, the project standard wins for that project.

## Usage Principles

- The global standard stores rules that are reusable across multiple projects.
- A project standard stores rules that are specific to one project.
- Rules promoted into the global standard should be stable, verifiable, and portable.
